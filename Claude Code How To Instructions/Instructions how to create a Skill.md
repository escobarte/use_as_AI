Вот аккуратно оформленная версия в `.md` формате:

````md
# Инструкция по созданию навыка (Skill)

Создание навыка происходит прямо в файловой системе вашего проекта без необходимости использовать API-ключи.

---

## Шаг 1: Создайте директорию навыка

В корне вашего проекта создайте папку для нового навыка:

```bash
mkdir -p .claude/skills/my-custom-skill
````

---

## Шаг 2: Создайте основной файл навыка

Внутри созданной папки создайте файл `SKILL.md`.
Это главный файл, который Claude будет читать.

---

## Шаг 3: Настройте параметры (Frontmatter)

В самом начале файла `SKILL.md` можно добавить YAML-блок с настройками.
Это опционально, но полезно для контроля:

```yaml
---
user-invocable: true # Позволяет вызывать навык командой /my-custom-skill
disable-model-invocation: false # Разрешает Claude использовать навык автоматически
---
```

---

## Шаг 4: Напишите инструкции

После Frontmatter опишите суть навыка на естественном языке.
Форматируйте текст с помощью Markdown. Старайтесь уложиться в 500 строк.

```md
# Описание

Ты — эксперт по нашей внутренней библиотеке компонентов.  
Используй этот навык всякий раз, когда пользователь просит создать новую кнопку или форму.

# Правила

1. Всегда используй классы из Tailwind.
2. Не используй стандартный тег `<button>`, импортируй `<CustomButton>` из `@/ui/button`.

# Примеры

Смотри примеры кода в файле `examples.ts` в этой же директории.
```

---

## Шаг 5: Добавьте вспомогательные файлы (Опционально)

В отличие от старых команд, папка Skills позволяет хранить дополнительные файлы рядом с `SKILL.md`, например:

* `examples.ts`
* API-справочники
* JSON-конфиги

Claude прочитает их, если вы сошлетесь на них в основном файле.

---

## Шаг 6: Использование

Теперь навык готов!

* Claude Code сам найдет его при запуске сессии
* Если навык актуален, он автоматически подгрузится в контекст
* Либо вы можете вызвать его вручную командой:

```bash
/my-custom-skill
```
