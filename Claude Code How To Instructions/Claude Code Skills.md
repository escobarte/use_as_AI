Claude Code Skills — это пользовательские расширения (модули с контекстом и инструкциями) для CLI-инструмента Claude Code. Это способ «научить» ИИ-агента специфике вашего проекта, используя открытый стандарт Agent Skills.

Среда использования - Основные сценарии:
* Генерация UI/Frontend: Настройка ИИ на использование вашей конкретной дизайн-системы или Tailwind-конфигов, чтобы избежать шаблонного «ИИ-дизайна»
* UI/UX Design  
* ?SEO ?
* Responsive Design & Cross‑Browser Compatibility 
* Frontend Development (HTML, CSS, JavaScript)  

Главная проблема типичной «ИИ-генерации» — это предсказуемость: карточки в три колонки, выровненный по центру текст, стандартные отступы и отсутствие глубины.
Прописать для Claude жесткие «анти-паттерны» (чего делать нельзя) и «креативные директивы» (что использовать обязательно).

# React + Tailwind CSS + Framer Motion.
```
Tailwind CSS: Это абсолютный лидер для ИИ. Claude знает его идеально. Вместо того чтобы писать сотни строк CSS в отдельном файле (что жрет токены на контекст), он пишет короткие классы прямо в HTML. Это позволяет быстро менять сетки, добавлять асимметрию и сложные градиенты.

Framer Motion: Лучшая библиотека для анимаций в React. Она декларативная. Чтобы сделать сложную анимацию появления элементов, Claude достаточно дописать пару атрибутов (initial, animate). Это выглядит премиально и очень лаконично в коде.

Lucide React: Пакет иконок. Они легкие, современные и Claude отлично умеет ими манипулировать.
```

Фреймворк	Next.js (React)	Четкая структура, минимум лишнего бойлерплейта.
Стили	Tailwind CSS	Минимум токенов, максимальная скорость правок.
Анимации	Framer Motion	Краткий синтаксис для создания «WOW-эффекта».
Типизация	TypeScript	Парадокс: требует больше токенов, но экономит их в разы, так как ИИ совершает меньше ошибок и реже переделывает код

## Как мы сэкономим токены в самом Скилле (Skill)?
Чтобы не скармливать Claude картинки при каждом запросе (это очень дорого), мы сделаем так:
    Вы один раз даете ему картинки.
    Он описывает их в текстовый файл «Visual_DNA.md» (стилистический код-бук).
    В дальнейшем он будет читать этот компактный текст вместо тяжелых изображений.

Плюшки:
Связь с вашими референсами: Добавьте в раздел Design Thinking строку: "Always check ./references/ folder for visual inspiration before deciding on the aesthetic direction".
Строгая адаптивность: В подразделе Spatial Composition добавьте: "Ensure fluid responsiveness using clamp() for typography and complex grid shifting for mobile/tablet breakpoints. Mobile version should not just stack elements, but reimagine the layout".

Ваш стек: Если мы решили использовать Next.js + Tailwind + Framer Motion, допишите это в Constraints, чтобы Клод не гадал, на чем писать.
